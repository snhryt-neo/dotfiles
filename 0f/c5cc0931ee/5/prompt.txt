Claude Code 上で /imagegen スラッシュコマンドとして呼び出せる画像生成コマンドを作りたいです。内部で Codex CLI の組み込み画像生成（$imagegen）に処理を委譲します。
ユーザーが /imagegen <プロンプト> を実行すると、Claude Code が codex exec で Codex CLI を呼び出し、gpt-image-2 で画像を生成します。生成された画像はカレントディレクトリ直下に保存し（デフォルトの ~/.codex/generated_images/ ではなく）、完了後に保存先パスを報告します。
スキルの種類は Claude Code カスタムスラッシュコマンドで、~/.claude/commands/imagegen.md に置いてグローバルに使えるようにしたいです。
前提として、Codex CLI はインストール済みで ChatGPT アカウントで認証済みです。API キーは使わず、Codex CLI の組み込み使用量で処理します。出力先の制御は codex exec への自然言語指示でまず試み、うまくいかない場合は find と mv のフォールバックで対応します。
インプットは以下を想定しています。プロンプト（日本語・英語どちらでも可）は必須です。オプションとして、サイズ（デフォルト 1024x1024）、quality（デフォルト medium）、出力ファイル名（デフォルトはプロンプトから自動生成した snake_case の PNG ファイル名）を指定できるようにしたいです。
テストケースのイメージは以下です。/imagegen 富士山と桜のポスター、/imagegen a minimalist logo for a data engineering tool、/imagegen dashboard mockup for BigQuery monitoring --size 1536x864、/imagegen 猫のアイコン --output cat_icon.png といった使い方を想定しています。

## Step 2 — 画像仕様の補完・確認

ユーザーのプロンプトをそのまま実行せず、まず画像生成に必要な仕様を補完する。

### 2.1 プロンプトの意図を分類

以下を推定する:

| 項目 | 例 |
|---|---|
| 用途 | アイコン、OGP、ポスター、SNS投稿、スライド挿絵、UIモック、壁紙 |
| 主題 | 人物、動物、風景、商品、図解、抽象表現 |
| テイスト | 写実、ミニマル、アニメ調、フラットイラスト、3D、手描き、水彩、サイバーパンク |
| 構図 | 正面、俯瞰、横長、中央配置、余白多め、クローズアップ |
| 文字入れ | あり / なし |
| 背景 | 単色、透明希望、自然背景、オフィス、抽象背景 |
| 禁止事項 | ロゴなし、文字なし、人物なし、過度な装飾なし |

### 2.2 不足情報の自動補完

ユーザーの指定が曖昧な場合、以下のデフォルトを使う:

| 項目 | デフォルト |
|---|---|
| テイスト | clean, modern, high quality |
| 構図 | main subject centered, balanced composition |
| 背景 | simple background |
| 文字 | 原則なし |
| 人物 | 指定がなければ出さない |
| ブランドロゴ | 指定がなければ出さない |

### 2.3 確認が必要なケース

以下のいずれかに該当する場合は、生成前にユーザーへ確認する:

- テイスト候補が複数ありそうな場合
- 用途に対してサイズが未指定の場合
- 文字入れの有無が成果物に大きく影響する場合
- 人物・顔・実在ブランド・ロゴが含まれる場合
- 透明背景を希望している場合
- 「いい感じに」「おしゃれに」など、解釈幅が大きい場合

確認メッセージ例:

```text
生成前に確認させてください。

1. テイスト: ミニマル / 写実 / フラットイラスト / 3D
2. 用途: アイコン / OGP / スライド挿絵 / その他
3. 文字入れ: あり / なし

未指定なら、ミニマルなフラットイラスト・文字なしで進めます。

prompt.md
